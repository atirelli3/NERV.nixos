## Directory Purposes

**`base/`:**
- Purpose: Complete deployable NixOS configuration for the primary workstation host (`nixos-base`)
- Contains: `flake.nix` (entry point), `configuration.nix` (machine-specific settings), `disko-configuration.nix` (disk provisioning)
- Key files: `base/flake.nix`, `base/configuration.nix`
- Note: `hardware-configuration.nix` is generated by `nixos-generate-config` at install time and is gitignored (not present in repo)

**`modules/`:**
- Purpose: Composable NixOS modules imported selectively by host flakes; each file owns exactly one concern
- Contains: 10 `.nix` files covering bluetooth, hardware, kernel hardening, nix daemon, SSH/security services, audio, printing, secureboot, system security tools, and shell environment
- Key files: `modules/secureboot.nix`, `modules/security.nix`, `modules/kernel.nix`

**`server/`:**
- Purpose: NixOS configuration for a server host; uses tmpfs root (impermanence pattern) with dedicated LVM volumes for `/nix`, `/persist`, `/home`
- Contains: `configuration.nix`, `disko-configuration.nix`
- Key files: `server/configuration.nix`
- Note: No `flake.nix` yet — host is work in progress

**`vm/`:**
- Purpose: Placeholder for a future VM host configuration
- Contains: `.gitkeep` only
- Generated: No

**`.template/`:**
- Purpose: Reference implementations demonstrating the full intended architecture (impermanence + tmpfs root + secureboot + TPM2). Not evaluated by any deployed flake. Used as copy-paste source when creating new hosts.
- Contains: 6 `.nix` files
- Key files: `.template/configuration.nix` (most complete reference), `.template/flake.nix`
- Committed: Yes

**`.planning/codebase/`:**
- Purpose: GSD codebase analysis documents for AI-assisted planning and execution
- Contains: ARCHITECTURE.md, STRUCTURE.md (and future docs)
- Generated: Yes (by GSD map-codebase command)
- Committed: Yes

## Key File Locations

**Entry Points:**
- `base/flake.nix`: Flake entry for the `nixos-base` host — run `nixos-rebuild switch --flake /path/to/repo/base#nixos-base`

**Machine Configuration:**
- `base/configuration.nix`: Boot, filesystems, networking, users, locale for the workstation host
- `server/configuration.nix`: Boot, filesystems, impermanence setup for the server host

**Disk Provisioning:**
- `base/disko-configuration.nix`: Workstation disk layout (LUKS + LVM, ext4 root)
- `server/disko-configuration.nix`: Server disk layout (LUKS + LVM, separate store/persist/home LVs)

**Security Modules:**
- `modules/secureboot.nix`: Lanzaboote bootloader, TPM2 LUKS sealing, two-phase first-boot systemd services
- `modules/security.nix`: AppArmor, auditd rules, ClamAV, AIDE config and daily integrity check timer
- `modules/kernel.nix`: Zen kernel, 15+ kernel params, sysctl hardening, filesystem/network/kernel protections
- `modules/openssh.nix`: sshd on port 2222, endlessh tarpit on 22, fail2ban with incremental bans

**Service Modules:**
- `modules/pipewire.nix`: PipeWire audio stack with low-latency config and AirPlay support
- `modules/bluetooth.nix`: BlueZ with PipeWire integration, OBEX file transfer, MPRIS media key proxy
- `modules/printing.nix`: CUPS with Gutenprint and mDNS discovery via Avahi
- `modules/zsh.nix`: System-wide Zsh with Starship prompt, fzf, eza, Nerd Fonts, Git/Nix aliases

**Infrastructure Modules:**
- `modules/hardware.nix`: Redistributable firmware, AMD microcode, fwupd, weekly fstrim
- `modules/nix.nix`: Flakes enabled, wheel-only daemon access, auto-GC (20d retention), store optimise, daily auto-upgrade

**Reference:**
- `.template/configuration.nix`: Most complete reference showing impermanence, full hardening, immutable users
- `.template/flake.nix`: Reference flake with impermanence input

## Naming Conventions

**Files:**
- Module files: lowercase with hyphens for multi-word names (`openssh.nix`, `secureboot.nix`)
- Configuration files: `configuration.nix` (host config), `disko-configuration.nix` (disk layout), `flake.nix` (flake entry)
- One file per concern — file name matches the NixOS service or subsystem it configures

**Directories:**
- Host directories: lowercase, named for the role (`base`, `server`, `vm`)
- Special directories: dot-prefixed for non-deployed content (`.template`, `.planning`)

**Disk Labels:**
- Pattern: `NIX<ROLE>` in uppercase — `NIXBOOT`, `NIXLUKS`, `NIXROOT`, `NIXSWAP`, `NIXSTORE`, `NIXPERSIST`, `NIXHOME`
- Labels must be kept in sync between `disko-configuration.nix` and `configuration.nix`

**NixOS Module Arguments:**
- Standard signature: `{ config, lib, pkgs, ... }:` — use `lib.mkForce` to override conflicting option values

## Where to Add New Code

**New host (e.g., laptop):**
- Create `laptop/` directory mirroring `base/` structure
- Primary files: `laptop/flake.nix`, `laptop/configuration.nix`, `laptop/disko-configuration.nix`
- Import desired modules from `../modules/` in the flake's `modules = [ ... ]` list
- Reference `.template/flake.nix` and `.template/configuration.nix` for impermanence setup

**New shared module:**
- Implementation: `modules/<concern>.nix`
- Follow signature `{ config, lib, pkgs, ... }: { ... }`
- Add to the relevant host flake's `modules = [ ... ]` list in `<host>/flake.nix`

**Extending an existing module:**
- Edit the corresponding `modules/<concern>.nix` directly — changes apply to all hosts that import it

**Host-only overrides:**
- Place overrides directly in `<host>/configuration.nix` using `lib.mkForce` where needed to resolve merge conflicts with module defaults

**Disk layout changes:**
- Edit `<host>/disko-configuration.nix` and keep labels in sync with `<host>/configuration.nix` `fileSystems`
