## Cross-Phase Integration

**Wired:** 14/16 integration paths verified end-to-end

### Fully Wired Flows

**Flow 1 — nerv.* options → modules → aggregators → flake:** Complete. Every option namespace (`nerv.hostname`, `nerv.locale.*`, `nerv.primaryUser`, `nerv.hardware.*`, `nerv.openssh.*`, `nerv.audio.enable`, `nerv.bluetooth.enable`, `nerv.printing.enable`, `nerv.zsh.enable`, `nerv.home.*`, `nerv.secureboot.enable`) is defined in the appropriate module, aggregated through `system/default.nix` or `services/default.nix`, collected in `modules/default.nix`, exported as `nixosModules.default` in `flake.nix`, and consumed by `nixosConfigurations.nixos-base`.

**Flow 3 — nerv.primaryUser → user groups → nerv.home.users → HM activation:** Complete. `configuration.nix` sets both `nerv.primaryUser = ["demon0"]` and `nerv.home.users = ["demon0"]`; identity.nix wires groups; home/default.nix generates home-manager.users via listToAttrs; stateVersion inherited dynamically from osConfig.system.stateVersion. (Note: the two lists are independent by design — documented but requires operator awareness when adding users.)

**Flow 4 — NIXLUKS cross-reference chain:** Documentation-complete. The NIXLUKS label string is identical across `disko-configuration.nix`, `boot.nix`, and `secureboot.nix`. Bidirectional cross-reference comments present in all three files per DOCS-04.

### Partial Flows (Tech Debt)

**Flow 2 — impermanence + secureboot → sbctl assertion:** The IMPL-02 assertion logic in `impermanence.nix:66` is correct (`lib.optional config.nerv.secureboot.enable`). However, neither `nerv.impermanence.enable` nor `nerv.secureboot.enable` is declared in `configuration.nix` — both remain at their `false` defaults with no explicit host declaration. The assertion path is never evaluated by the current configuration.

**disko operational gap:** `hosts/nixos-base/disko-configuration.nix` is not imported anywhere in the NixOS evaluation graph, and `disko` is not a flake input. The disk partitioning step must be run manually before first installation. This is a deployment procedure gap, not a module evaluation failure, but an operator following only `nixos-rebuild build` will not have the NIXLUKS partition that `boot.nix` and `secureboot.nix` depend on.

## Tech Debt

### Total: 5 items across 3 areas

**1. Legacy flat modules/ files (9 files) — cleanup needed**

All of the following are superseded by the modules/system/ and modules/services/ subtrees and are not imported by any aggregator. They are harmless today but create maintenance confusion:

- `modules/openssh.nix` → superseded by `modules/services/openssh.nix`
- `modules/pipewire.nix` → superseded by `modules/services/pipewire.nix`
- `modules/bluetooth.nix` → superseded by `modules/services/bluetooth.nix`
- `modules/printing.nix` → superseded by `modules/services/printing.nix`
- `modules/zsh.nix` → superseded by `modules/services/zsh.nix` (also contains stale `/etc/nixos#nixos` aliases)
- `modules/kernel.nix` → superseded by `modules/system/kernel.nix`
- `modules/security.nix` → superseded by `modules/system/security.nix`
- `modules/nix.nix` → superseded by `modules/system/nix.nix`
- `modules/hardware.nix` → superseded by `modules/system/hardware.nix`

Documented in Phase 02-03 and Phase 03-03 SUMMARYs as pending cleanup.

**2. Unused impermanence flake input**

`flake.nix` declares `impermanence` as an input and receives it in the `outputs` function signature, but `impermanence.nixosModules.impermanence` is never added to any `nixosConfigurations` modules list. The custom `modules/system/impermanence.nix` is self-contained (documented decision: "upstream nixos-community/impermanence module is NOT used"). The input could be removed from `flake.nix`, or a comment should explain its presence. Currently causes a dead entry in `flake.lock`.

**3. disko not wired as a flake input**

`disko` is not declared in `flake.nix` inputs. `disko-configuration.nix` must be run as a standalone command before initial installation. For v2.0, consider adding `disko` as a flake input and including `disko.nixosModules.disko` + importing `./hosts/nixos-base/disko-configuration.nix` in `nixosConfigurations.nixos-base` to enable declarative disk management via `nixos-rebuild`.

**4. IMPL-02 sbctl assertion never exercised**

`nerv.impermanence.enable` and `nerv.secureboot.enable` should be explicitly declared in `configuration.nix` (even as `= false`) to make the intent visible and to document the activation path for operators who need persistence + secureboot.

**5. Nyquist validation not completed for any phase**

All 6 VALIDATION.md files are in `draft` status with `nyquist_compliant: false`. No phase has automated test coverage. This is acceptable for a configuration-as-code project where the primary verification is `nixos-rebuild build`, but the VALIDATION.md drafts should be either completed or explicitly closed before v2.0.
