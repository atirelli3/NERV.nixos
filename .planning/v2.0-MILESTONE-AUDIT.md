## Issues Detail

### 1. CRITICAL — `nixosConfigurations.server` missing (PROF-02, DISKO-02)

`flake.nix` has a single `let` binding (`host`) and a single `nixosConfigurations` entry. The `server` profile and configuration were never wired. The entire LVM code path in `disko.nix` and `full` mode in `impermanence.nix` are unreachable dead code.

**Fix:** Add `server = { nerv.disko.layout = "lvm"; nerv.impermanence.mode = "full"; ... };` let-binding and corresponding `nixosConfigurations.server` mirroring the `host` structure.

### 2. HIGH — @root-blank step absent from README BTRFS walkthrough (PROF-04, BOOT-02)

The rollback service executes `btrfs subvolume snapshot /btrfs_tmp/@root-blank /btrfs_tmp/@` at every boot. If `@root-blank` doesn't exist (not created during install), the first boot produces an unbootable system.

**Fix:** Insert `sudo btrfs subvolume snapshot -r /mnt/@ /mnt/@root-blank` as a mandatory step between disko provisioning and nixos-install in the README BTRFS section.

### 3. MEDIUM — Profiles cross-reference comments absent (PROF-03)

`disko.nix`, `boot.nix`, and `impermanence.nix` headers contain no `# Profiles :` cross-reference lines despite Phase 12-02 claiming otherwise.

**Fix:** Append `# Profiles : hostProfile → layout = "btrfs" | serverProfile → layout = "lvm"` style comments to the three module file headers.

### 4. LOW — Stale comment in `modules/system/default.nix` (PROF-03)

Line 13: `# declarative disk layout — conditional LVM LVs based on impermanence mode` — stale since Phase 9; conditioning is now on `nerv.disko.layout`.

**Fix:** Update to `# declarative disk layout (btrfs/lvm) with layout-conditional initrd services`.
