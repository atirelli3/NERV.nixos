  <name>Task 2: Add btrfs lib.mkIf block with neededForBoot, environment.persistence, and sbctl warning</name>

  <action>
    Append a third entry to the lib.mkMerge list in the config section. After the closing `})` of the existing full-mode block (currently the last entry), add:

    ```nix
    (lib.mkIf (cfg.mode == "btrfs") {
      # /persist must be available before impermanence bind mounts execute in stage 2.
      # disko v1.13.0 does not support neededForBoot on subvolume mounts (issues #192, #594
      # closed as "use fileSystems directly"); this override is the canonical approach.
      fileSystems."${cfg.persistPath}".neededForBoot = lib.mkDefault true;

      # environment.persistence — bind mounts from /persist for desktop/laptop BTRFS mode.
      # Root is reset to @root-blank on every boot (Phase 10 rollback service), so /var/lib
      # is empty at stage 2 activation — no bind-over-content risk.
      # Requires impermanence.nixosModules.impermanence in nixosConfigurations modules list.
      environment.persistence."${cfg.persistPath}" = {
        hideMounts = true;
        directories = [
          "/var/lib"   # all service state: uid/gid (nixos), timers (systemd), sbctl, BT, NM, cups...
          "/etc/nixos" # NERV.nixos repo — user clones here, must survive rollback
          # NOTE: /var/log intentionally omitted — persisted by @log BTRFS subvolume (disko.nix).
          #       Adding it here would create a double-mount conflict at stage-2 activation.
        ];
        files = [
          "/etc/machine-id"                    # stable machine identity (journald, systemd)
          "/etc/ssh/ssh_host_ed25519_key"      # SSH host identity — must survive rollback
          "/etc/ssh/ssh_host_ed25519_key.pub"
          "/etc/ssh/ssh_host_rsa_key"
          "/etc/ssh/ssh_host_rsa_key.pub"
        ];
      };

      # Belt-and-suspenders: /var/lib as a single broad entry already covers /var/lib/sbctl.
      # This warning fires only if someone strips the persistence list and forgets sbctl.
      # Using lib.warn (not assertion) — preserves nix flake check during multi-step migrations;
      # missing sbctl persistence loses keys on next rollback but is recoverable via re-enrollment,
      # unlike the IMPL-02 scenario (tmpfs wipe) which uses a hard assertion.
      warnings = lib.optionals
        (config.nerv.secureboot.enable &&
         !(lib.any
             (d: d == "/var/lib" || d == "/var/lib/sbctl" || lib.hasPrefix "/var/lib/sbctl" d)
             (map (e: if builtins.isString e then e else e.directory or "")
               (environment.persistence."${cfg.persistPath}".directories or []))))
        [ "nerv: secureboot is enabled but /var/lib/sbctl is not covered by environment.persistence in btrfs mode — sbctl keys will be lost on rollback. Add \"/var/lib\" or \"/var/lib/sbctl\" to the persistence directories." ];
    })
    ```

    IMPORTANT implementation note on the warnings check: The `environment.persistence` attrset is accessed inside the same `lib.mkIf` block. To avoid self-reference, simplify the sbctl check to inspect `cfg` options directly rather than the built attribute. Use this simpler form that is free of self-reference cycles:

    ```nix
    warnings = lib.optionals
      (config.nerv.secureboot.enable &&
       !(lib.any
           (d: d == "/var/lib" || d == "/var/lib/sbctl" || lib.hasPrefix "/var/lib/sbctl" d)
           ["/var/lib" "/etc/nixos"]))  # static list mirrors the locked persistence dirs above
      [ "nerv: secureboot is enabled but /var/lib/sbctl is not covered..." ];
    ```

    Since the btrfs persistence directories are fixed in this module (locked decision: /var/lib is always included), the static check `lib.any (d: d == "/var/lib") ["/var/lib" "/etc/nixos"]` will always be true, meaning the warning will never fire under the current configuration. This is correct: the warning is a guard for future edits that remove /var/lib from the list. Write the check to inspect the actual `environment.persistence."${cfg.persistPath}".directories` value — but since that would be a self-reference, instead hardcode the static list of directories being set in this same block and check that list. The simplest correct implementation:

    ```nix
    warnings =
      let
        persistDirs = [ "/var/lib" "/etc/nixos" ];
        sbctlCovered = lib.any
          (d: d == "/var/lib" || d == "/var/lib/sbctl" || lib.hasPrefix "/var/lib/sbctl" d)
          persistDirs;
      in
        lib.optionals (config.nerv.secureboot.enable && !sbctlCovered)
          [ "nerv: secureboot is enabled but /var/lib/sbctl is not covered by environment.persistence in btrfs mode — sbctl keys will be lost on rollback." ];
    ```

    After adding the block, run `nix flake check` to confirm no errors. Then run the two manual eval spot-checks and record their output in the plan summary.
